==============================================================================
Configuration Reference                                *NeoAI-config-reference*

The configuration is managed by:
  - default_config.lua: Default values
  - merger.lua: Validation and merging of user config with defaults
  - keymap_manager.lua: Keymap CRUD and persistence
  - state.lua: Coroutine context state manager

==============================================================================

==============================================================================
Configuration Validation                                    *NeoAI-config-validate*

The merger.lua module validates user configuration:

Type checking:
  - Each field is checked against expected Lua types
  - Mismatched types are rejected with a warning notification
  - Rejected fields fall back to default values

Enum validation:
  - ui.default_ui: "tree", "chat"
  - ui.window_mode: "float", "tab", "split"
  - log.level: "DEBUG", "INFO", "WARN", "ERROR", "FATAL"

Range validation:
  - window.width >= 10
  - window.height >= 5
  - split.size >= 10
  - max_history_per_session >= 1
  - log.max_file_size >= 1024
  - log.max_backups >= 0
  - test.delay_ms >= 0

Free-form tables (no strict validation):
  - ai.providers: Provider definitions
  - ai.scenarios: Scenario configurations
  - keymaps: Keymap configurations
  - tools.approval.tool_overrides: Tool-specific approval overrides

Keymap persistence:
  - Keymaps are saved to stdpath("config") .. "/neoai_keymaps.json"
  - Only non-default keymaps are saved
  - Loaded on initialization via keymap_manager.load_keymaps()

==============================================================================
Config Module Architecture                               *NeoAI-config-modules*

config/init.lua:
  Entry point, initializes all config sub-modules

config/merger.lua:
  - process_config(user_config): Validate, merge, and return complete config
  - get_scenario_candidates(scenario): Get AI candidates for a scenario
  - get_preset(scenario): Get first available AI preset
  - get_available_models(): Get all models with valid API keys

config/keymap_manager.lua:
  - initialize(config): Load default keymaps and persisted overrides
  - get_keymap(context, action): Get keymap for specific action
  - set_keymap(context, action, key, desc): Set custom keymap
  - save_keymaps() / load_keymaps(): Persist to JSON file
  - export_formatted(): Return formatted keymap display

config/state.lua:
  - create_context(ctx): Create coroutine-local context
  - with_context(ctx, fn): Execute function within context
  - get_shared(): Get shared state for current coroutine
  - get_current_context(): Get current coroutine context
